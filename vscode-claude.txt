# Claude Code CLI 交互手册

基于 VSCode Claude Code 插件分析

## 1. CLI 启动参数

### 基本参数
```javascript
const args = [
  '--print',                    // 非交互模式
  '--verbose',                  // 详细输出
  '--output-format', 'stream-json',  // JSON 流式输出
  '--input-format', 'stream-json',   // JSON 流式输入
]
```

### 权限相关参数

#### 方式一：使用 canUseTool 回调（推荐）
```javascript
// 当提供 canUseTool 回调时，自动添加 --permission-prompt-tool stdio
if (canUseToolCallback) {
  args.push('--permission-prompt-tool', 'stdio')
}
```

#### 方式二：手动指定
```javascript
// 手动指定权限提示工具
args.push('--permission-prompt-tool', 'stdio')
```

#### 其他权限参数
```javascript
'--permission-mode', 'default'    // 权限模式: acceptEdits, bypassPermissions, default, delegate, dontAsk, plan
'--allow-dangerously-skip-permissions'  // 跳过所有权限检查
'--dangerously-skip-permissions'        // 同上
```

### 其他常用参数
```javascript
'--model', 'sonnet'              // 指定模型
'--max-turns', '100'              // 最大回合数
'--continue'                      // 继续对话
'--resume', sessionId             // 恢复会话
'--allowedTools', 'Bash,Read'     // 允许的工具
'--disallowedTools', 'Write'      // 禁止的工具
'--add-dir', '/path'              // 允许访问的目录
'--mcp-config', jsonString        // MCP 服务器配置
'--session-id', uuid               // 指定会话 ID
'--no-session-persistence'        // 禁用会话持久化
'--include-partial-messages'      // 包含部分消息
'--thinking', 'adaptive'          // 思考模式: enabled, disabled, adaptive
'--debug'                         // 调试模式
```

## 2. 消息格式

### 2.1 消息类型 (Message Types)

#### 发送的消息格式
所有发送到 CLI 的消息都需要用换行符 `\n` 结尾：
```javascript
const message = {
  type: 'user',  // 或 'control_request'
  message: { ... }
}
const jsonMessage = JSON.stringify(message) + '\n'
stdin.write(jsonMessage)
```

#### 接收的消息类型
CLI 会发送以下类型的消息：
- `system` - 系统初始化消息
- `assistant` - Claude 响应
- `user` - 用户消息（输入回显）
- `tool_use` - 工具调用请求
- `tool_result` - 工具执行结果
- `control_request` - 控制请求（如权限请求）
- `control_response` - 控制响应
- `control_cancel_request` - 取消请求
- `result` - 最终结果
- `keep_alive` - 保活消息
- `streamlined_text` - 精简文本
- `streamlined_tool_use_summary` - 精简工具调用摘要

## 3. 权限处理 (control_request / control_response)

### 3.1 接收权限请求 (control_request)

当 CLI 需要权限时，会发送 `control_request` 消息：

```javascript
// 接收到的消息格式
{
  "type": "control_request",
  "request_id": "uuid-xxx",
  "request": {
    "subtype": "can_use_tool",           // 请求子类型
    "tool_name": "Read",                  // 工具名称
    "input": { "file_path": "..." },      // 工具输入参数
    "permission_suggestions": [
      {
        "type": "addRules",
        "rules": [{ "toolName": "Read", "ruleContent": "//path/**" }],
        "behavior": "allow",
        "destination": "session"
      }
    ],
    "decision_reason": "Path is outside allowed working directories",
    "tool_use_id": "call_function_xxx",
    "agent_id": "..."                    // 可选，agent ID
  }
}
```

### 3.2 发送权限响应 (control_response)

**重要**: `request_id` 必须放在 `response` 对象内部，并且需要 `subtype` 字段！

#### 正确格式：
```javascript
// 允许权限
const response = {
  type: 'control_response',
  response: {
    subtype: 'success',
    request_id: 'uuid-xxx',
    response: {
      approved: true
      // 或其他响应内容
    }
  }
}

// 拒绝权限
const response = {
  type: 'control_response',
  response: {
    subtype: 'error',
    request_id: 'uuid-xxx',
    error: 'Permission denied by user'
  }
}
```

#### 错误格式（不要使用）：
```javascript
// ❌ 错误 - request_id 在顶层
{
  type: 'control_response',
  request_id: 'uuid-xxx',  // 错误位置！
  response: { approved: true }
}

// ❌ 错误 - 缺少 subtype
{
  type: 'control_response',
  response: {
    request_id: 'uuid-xxx',  // 缺少 subtype
    response: { approved: true }
  }
}
```

## 4. control_request 子类型 (subtype)

### 4.1 can_use_tool
请求使用工具的权限。

**请求**:
```javascript
{
  request: {
    subtype: 'can_use_tool',
    tool_name: 'Read',
    input: { file_path: '/path/to/file' },
    permission_suggestions: [...],
    decision_reason: 'Path is outside allowed working directories',
    tool_use_id: 'call_function_xxx'
  }
}
```

**响应**:
```javascript
{
  response: {
    subtype: 'success',
    request_id: 'xxx',
    response: {
      approved: true,  // 或 false
      // 可选：添加规则
      // { type: 'addRules', rules: [...], behavior: 'allow', destination: 'session' }
    }
  }
}
```

### 4.2 hook_callback
处理 Hook 回调。

**请求**:
```javascript
{
  request: {
    subtype: 'hook_callback',
    callback_id: 'hook_xxx',
    input: { ... },
    tool_use_id: 'call_function_xxx'
  }
}
```

### 4.3 mcp_message
MCP 服务器消息。

**请求**:
```javascript
{
  request: {
    subtype: 'mcp_message',
    server_name: 'mcp-server-name',
    message: { method: '...', params: ..., id: ... }
  }
}
```

### 4.4 elicitation
MCP 交互请求。

**请求**:
```javascript
{
  request: {
    subtype: 'elicitation',
    mcp_server_name: 'server-name',
    message: { ... },
    mode: '...',
    url: '...',
    elicitation_id: '...',
    requested_schema: { ... }
  }
}
```

### 4.5 其他子类型

这些通过 `control_request` 的 `request` 字段发送：

```javascript
// 初始化
{ subtype: 'initialize', hooks: {...}, sdkMcpServers: [...], ... }

// 停止任务
{ subtype: 'stop_task', task_id: 'xxx' }

// 中断
{ subtype: 'interrupt' }

// 设置权限模式
{ subtype: 'set_permission_mode', mode: 'default' }

// 设置模型
{ subtype: 'set_model', model: 'sonnet' }

// 设置最大思考 token 数
{ subtype: 'set_max_thinking_tokens', max_thinking_tokens: 1024 }

// 应用标志设置
{ subtype: 'apply_flag_settings', settings: {...} }

// 回滚文件
{ subtype: 'rewind_files', user_message_id: 'xxx', dry_run: true/false }

// 远程控制
{ subtype: 'remote_control', enabled: true/false }

// MCP 重连
{ subtype: 'mcp_reconnect', serverName: 'xxx' }

// MCP 开关
{ subtype: 'mcp_toggle', serverName: 'xxx', enabled: true/false }

// MCP 认证
{ subtype: 'mcp_authenticate', serverName: 'xxx' }

// MCP 清除认证
{ subtype: 'mcp_clear_auth', serverName: 'xxx' }

// MCP OAuth 回调
{ subtype: 'mcp_oauth_callback_url', serverName: 'xxx', callbackUrl: 'xxx' }

// MCP 状态
{ subtype: 'mcp_status' }

// MCP 设置服务器
{ subtype: 'mcp_set_servers', servers: {...} }
```

## 8. 实现示例

### 8.1 完整的实现框架

```javascript
const { spawn } = require('child_process')
const readline = require('readline')

class ClaudeCLIController {
  constructor() {
    this.process = null
    this.pendingCallbacks = new Map()
  }

  start() {
    const args = [
      '--print',
      '--verbose',
      '--output-format', 'stream-json',
      '--input-format', 'stream-json',
      '--permission-prompt-tool', 'stdio'
    ]

    this.process = spawn('claude', args, {
      cwd: process.cwd(),
      env: { ...process.env },
      stdio: ['pipe', 'pipe', 'pipe']
    })

    this.setupOutputHandlers()
  }

  setupOutputHandlers() {
    const rl = readline.createInterface({
      input: this.process.stdout
    })

    rl.on('line', (line) => {
      if (!line.trim()) return
      const message = JSON.parse(line)
      this.handleMessage(message)
    })

    this.process.stderr.on('data', (data) => {
      console.error('[stderr]', data.toString())
    })

    this.process.on('exit', (code) => {
      console.log('[exit]', code)
    })
  }

  handleMessage(message) {
    switch (message.type) {
      case 'control_request':
        this.handleControlRequest(message)
        break
      case 'assistant':
        // 处理 assistant 消息
        break
      case 'system':
        // 处理系统消息
        break
      case 'result':
        // 处理结果
        break
    }
  }

  async handleControlRequest(message) {
    const { request_id, request } = message

    if (request.subtype === 'can_use_tool') {
      // 处理权限请求
      // 显示 UI 让用户确认...

      // 发送响应
      this.sendControlResponse(request_id, true, {})
    }
  }

  sendControlResponse(requestId, approved, response = {}) {
    const message = {
      type: 'control_response',
      response: {
        subtype: 'success',
        request_id: requestId,
        response: {
          approved: approved,
          ...response
        }
      }
    }
    this.sendMessage(message)
  }

  sendToolResult(toolUseId, content, isError = false) {
    const message = {
      type: 'user',
      message: {
        role: 'user',
        content: [
          {
            type: 'tool_result',
            tool_use_id: toolUseId,
            content: content,
            is_error: isError
          }
        ]
      }
    }
    this.sendMessage(message)
  }

  sendMessage(message) {
    const jsonMessage = JSON.stringify(message) + '\n'
    this.process.stdin.write(jsonMessage)
  }

  sendUserMessage(text) {
    const message = {
      type: 'user',
      message: {
        role: 'user',
        content: [
          { type: 'text', text: text }
        ]
      }
    }
    this.sendMessage(message)
  }
}
```

## 9. 常见问题

### Q1: 为什么发送 control_response 后没有响应？
检查消息格式是否正确：
- `request_id` 必须在 `response` 对象内部
- 必须包含 `subtype` 字段

### Q2: 如何处理多个权限请求？
使用 `request_id` 跟踪每个请求的响应。

### Q3: 工具结果应该发送给谁？
工具结果需要发送给 `tool_use_id` 对应的工具调用。

### Q4: 如何取消正在执行的工具？
发送 `control_cancel_request`:
```javascript
{
  type: 'control_cancel_request',
  request_id: '原始请求的request_id'
}
```
